# Composition Patterns and Originality Rules

## Base Structure Types

Choose one structure per image — don't mix too many.

### Workflow

Best for: Input -> Processing -> Output, content production, AI workflows, automation chains.

Drawing: Input on the left, Xiaohei or quirky machine processing in the middle, output on the right. Orange arrows show main flow.

### System Component

Best for: Information sources, filters, databases, renderers, agent subsystems.

Drawing: Only 3-5 core modules. Xiaohei participates in one key action.

### Before/After

Best for: Chaos/Order, Manual/Automatic, Scattered/Collected, Anxious/Stable.

Drawing: Chaos on the left, stability on the right, orange arrow in between. Characters can be more exaggerated.

### Role State

Best for: User pain points, creator states, tool overload, information anxiety, stuck to flowing.

Drawing: 2-4 small states, each with a short label.

### Concept Metaphor

Best for: Content factories, information warehouses, black-box brains, workflow machines, auto-reports.

Drawing: One large quirky machine or object, few inputs, one output. Must be memorable.

### Method Layering

Best for: Methodology frameworks, system levels, capability stacks, content system layers.

Drawing: Layered boxes (not a formal pyramid). Xiaohei carrying bricks or building on the side.

### Map/Route

Best for: Idea to launch, user paths, content handoff paths, learning journeys.

Drawing: One winding path, few nodes, Xiaohei pulling a thread or walking.

### Comic Strip

Best for: Failure to success, real processes, rants, before/after usage changes.

Drawing: 2-4 small scenes, each frame shows one action only.

## Original Metaphor Generation Method

Always invent fresh metaphors from the current article. Never copy old images.

### Three Steps

1. Replace the abstract concept with a physical action: stuck, leaked, heavy, sorted, settled, fermented, opened, folded, unpacked, flowed back.
2. Replace the system structure with a low-tech object: broken machine, cardboard box, drawer, water pipe, mailbox, quirky dial, scale, well, ladder, weird workstation.
3. Let Xiaohei perform the action: not standing aside, but stuck in the machine, pulling the wrong wire, guarding the door, carrying, patching, weighing, holding the ladder, recording, stuffing something into a strange device.

### Object Pool

- Cardboard box, drawer, old machine, funnel, scale, mailbox, door, well, ladder, water pipe, ball of yarn, gate, turntable, black box, hole punch, pasta maker, clothesline, weird workstation.
- Use only 1-2 per image — don't crowd.

### Xiaohei Action Pool

- Pull, carry, stuff, fish out, press, weigh, sew, cut, twist, guard, push, catch, dismantle, mark, recycle.
- Actions must serve the core meaning — don't be strange for the sake of strange.

## Anti-Copy Rules

Don't open or replicate `assets/examples/` by default. These images are only for style calibration: line density, whitespace, color restraint, Xiaohei's vibe.

Unless the user explicitly says "copy this one / replicate this composition / use this example as a base", don't directly reuse these old compositions:

- Conveyor belt with two breakpoints
- Xiaohei pulling a judgment lever inside a content machine
- Xiaohei as a funnel sorting traffic/trust/conversion
- Xiaohei slicing a material fish
- Xiaohei guiding a handoff path
- Xiaohei pulling three layers of information sources
- Three Xiaohei characters holding megaphone/building bridge/opening door
- Xiaohei stamping a handoff phrase toolbox
- Xiaohei holding a sign watching common pitfall paths

Even for similar topics, invent new metaphors. For example, "handoff path" doesn't have to be a route — it could be Xiaohei connecting a content tail to a door handle. "Multi-use" doesn't have to involve fish — it could be Xiaohei pressing a paper ball into multiple shapes.
